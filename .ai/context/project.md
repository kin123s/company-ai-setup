## 3. 디렉토리 구조 (Directory Structure)

```text
/
├── .ai/                  # AI 에이전트 전역 규칙 및 컨텍스트, 상태 세션 관리 보관소
│   ├── rules.md          # 전역 코딩/작업 룰
│   ├── context/          # 프로젝트 아키텍처 및 도메인 지식
│   ├── workflows/        # 반복 작업(배포, 빌드, 테스트 등) 가이드라인
│   ├── guidelines/       # 세부 가이드라인 (에이전트 연동, 코딩 컨벤션 등)
│   ├── roles/            # 특정 에이전트 역할별 프롬프트
│   └── state/sessions/   # AI 작업 세션 핸드오프용 상태 기록 (매우 중요)
├── src/                  # 메인 소스코드 (프론트/백엔드 등에 따라 하위 구조 구성)
├── docs/                 # 기획서, API 스펙 등 공식 문서
└── [기타 디렉토리들]
```
