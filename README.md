# 🏢 Company AI Setup Template (AI 전역 워크스페이스 아키텍처)

> **사내 AI 코딩 어시스턴트(Cursor, Copilot, Antigravity 등) 도입 및 활용을 위한 전사 표준 워크스페이스 아키텍처 템플릿입니다.**

현대의 엔터프라이즈 개발 환경에서 AI 에이전트의 활용은 생산성 향상을 위해 필수적입니다. 하지만 개별 프로젝트마다 에이전트의 프롬프트가 파편화되고, 작업 컨텍스트가 초기화되며, 사내 보안 규정이나 유지보수 컨벤션이 무시된 코드가 무분별하게 생성되는 등 구조적인 제어 문제가 발생할 수 있습니다.

본 프로젝트는 단순한 코드 저장소가 아니라, **AI 툴들이 사내 인프라 규정, 코딩 컨벤션, 작업 히스토리를 중앙 집중적으로 인지하고 개발자와 안전하게 협업할 수 있도록 설계된 '시스템 설계서(System Architecture)'**입니다.

## 🌟 핵심 목표 (Core Objectives)

1. **컨텍스트 유지 (Context Preservation)**
   - AI 에이전트의 세션을 파일 기반으로 관리하여, 다단계 작업이나 담당자 변경 시에도 프로젝트 맥락 유실을 방지하고 완벽한 핸드오프(Handoff) 지원.
2. **거버넌스와 보안 (Governance & Security)**
   - 제로 트러스트(Zero-Trust) 아키텍처, 네트워크 리소스 보안, 사내 볼륨 구조(SSD/NAS 분리) 등 엔터프라이즈 환경의 인프라 정책 준수 강제.
3. **일관된 워크플로우 (Consistent Workflow)**
   - 빌드, 테스트, 배포 등 반복적인 DevOps 파이프라인에 대해 사람과 AI가 동일하게 밟아갈 수 있는 마크다운 기반의 실행 가이드라인 제공.
4. **생산성 극대화 (Productivity)**
   - 단일 프롬프트 주입만으로 신규 프로젝트에 AI 환경을 즉시 구성(Auto-bootstrapping)하여 온보딩 리소스의 획기적 단축.

## 📁 디렉토리 아키텍처 (Directory Architecture)

```text
/
├── .ai/                    # AI 에이전트 전역 규칙 및 컨텍스트, 상태 세션 관리 보관소 (Core System)
│   ├── rules.md            # 전역 코딩/작업 룰 (보안 정책, 인프라 규칙, 언어별 컨벤션 등 강제화)
│   ├── context/            # 프로젝트 아키텍처, 도메인 지식, 기술 스택 요약 문서
│   ├── workflows/          # 개발 주기(CI/CD, 빌드 등)에 따른 반복 작업 가이드라인
│   ├── guidelines/         # 에이전트 연동 방법론 등 세부 가이드
│   ├── roles/              # 인프라, 백엔드 등 특정 관점의 임무를 수행할 에이전트 역할별 프롬프트
│   └── state/sessions/     # AI 작업 세션 핸드오프 및 상태 기록용 (Context 유지의 핵심)
├── AGENT_PROMPT.md         # 신규 프로젝트에 본 환경을 자동 구축하기 위한 부트스트래핑 초기 프롬프트
└── ...
```

## 🚀 도입 효과 (Expected Benefits)

- **기술 부채(Technical Debt) 방지:** 규격화되지 않은 AI 생성 코드로 인한 시스템 레거시화 및 유지보수성 저하를 원천 차단.
- **협업 효율성 증대:** AI 에이전트가 강제적으로 생성하는 일관된 작업 인수인계서(Handover Report)를 통해 작업 이력의 완전한 추적 및 휴먼 에러 방지.
- **시니어 관점의 시스템 유지:** 주니어 개발자나 타 부서 인력이 AI를 사용하더라도, 프로젝트를 관통하는 시니어/아키텍트의 시스템 설계 의도를 잃지 않도록 통제.

## 🛠 사용 방법 (How to Use)

신규 작업 환경(레포지토리) 구성 시, 초기화 안내 프롬프트인 `AGENT_PROMPT.md`의 내용을 기반으로 AI 에이전트에게 전달하여 즉각적으로 `.ai` 아키텍처 기반을 복제 및 구성할 수 있습니다.
